# Phase 17 Completeness Pass

Phase 17 adds restore/extraction support on top of the Phase 16 verifier.

Implemented:

- CLI extraction mode via `--extract ARCHIVE.zip --output-dir DIR`.
- Validation of unsupported extraction combinations:
  - rejects positional output/input paths in extraction mode;
  - rejects `--verify`, `--manifest`, `--ignore`, `--prefix`, `--compression`, size limits, deterministic creation mode, and `--symlinks=follow` with extraction;
  - allows `--dry-run`, `--list-json`, and `--symlinks=skip|store-link`.
- Workflow integration through `Backup.Restore` and `Execution_Restore_Failed`.
- Restore implementation that first runs the Phase 16 archive verifier, preserving validation of:
  - ZIP local and central-directory structures;
  - ZIP64 records;
  - compression methods 0 and 8;
  - CRC32 and size metadata;
  - duplicate archive paths;
  - archive path invariants and backslash rejection;
  - manifest consistency when a manifest is present;
  - symlink metadata produced by Phase 15.
- Safe extraction planning:
  - normalized archive paths are mapped under the configured output directory only;
  - parent directories are created as needed for restored files;
  - existing destination paths are refused by default;
  - manifest entries are validated but not written as restored payload files;
  - symlink entries are skipped by default with explicit diagnostics;
  - `--symlinks=store-link` validates link targets and reports unsupported runtime symlink creation rather than silently writing unsafe links.
- Payload restoration:
  - stored entries are copied from validated local payload bytes;
  - method-8 entries generated by this project are inflated from deterministic raw deflate stored blocks;
  - payload location is re-derived from validated local headers.
- Reporting:
  - human extraction report;
  - deterministic JSON extraction report with `backup-restore-v1` format marker;
  - dry-run extraction actions (`would-restore`, `would-skip`) without creating the destination directory.
- Test coverage added in `backup_restore_tests.adb` for:
  - stored archive extraction;
  - deflated archive extraction;
  - existing destination refusal;
  - dry-run JSON output;
  - default symlink skip policy;
  - traversal archive rejection through verification-backed extraction;
  - missing `--output-dir` diagnostics.

Notes:

- The verifier remains the canonical ZIP integrity gate. Restore does not write any payload until verification succeeds.
- This environment did not provide `gprbuild`, so I could not execute the Ada test suite here.

Completeness pass updates:

- Fixed extraction of zero-byte stored entries by avoiding unsigned/offset underflow when the validated compressed payload length is zero.
- Added extraction-time CRC32 and uncompressed-size validation over the exact bytes written during restore. The verifier remains the front-door integrity gate, but restore now independently checks the restored byte stream before reporting success.
- Added restore tests for:
  - zero-byte stored file extraction;
  - manifest-bearing archive extraction with manifest validation but no manifest payload restoration;
  - unsafe symlink target rejection when link restoration is explicitly requested;
  - test-root cleanup for repeatable local test execution.

## Completeness pass 2

Additional restore hardening completed:

- Added `Action_Would_Reject` so dry-run extraction can report entries that would be rejected without mutating the target tree or failing the dry-run command.
- Changed dry-run behavior for existing destination paths from a hard restore failure to deterministic `would-reject` reporting with the original refusal reason.
- Changed dry-run behavior for unsafe or unsupported symlink restoration under `--symlinks=store-link` to deterministic `would-reject` reporting.
- Added partial-output cleanup for extraction failures after a destination file has been created, including deflate decode failure, unexpected unsupported method after verification, CRC/size mismatch, and write exceptions.
- Added regression coverage for dry-run existing-destination reporting and dry-run unsafe-symlink rejection reporting.

Build/test note: this environment still lacks a GNAT Ada compiler driver (`gprbuild`, `gnatmake`, and `gnat1` are unavailable), so this pass was completed by static inspection and package-level patching only.

## Completeness pass 3

Additional restore path-safety hardening completed:

- Added explicit detection of existing symbolic-link components in the configured `--output-dir` path before creating or writing into the target tree.
- Added explicit detection of existing symbolic-link ancestor directories for each restored archive entry before parent-directory creation and file creation.
- Existing symlinked destination parents now produce a precise restore-target diagnostic instead of allowing writes to be redirected outside the selected target tree.
- Dry-run extraction classifies entries whose destination ancestors are symlinks as `would-reject`, preserving deterministic reporting without mutating the filesystem.
- Added regression coverage for refusing extraction through a symlinked destination parent and refusing a symlinked output directory.

Build/test note: `gprbuild`, `gnatmake`, and `gnat1` remain unavailable in this environment, so this pass was completed by static inspection and source-level patching only.

## Completeness pass 4

Additional restore target-handling hardening completed:

- Changed `--output-dir` inspection/creation failures from a generic internal restore failure into an explicit `Restore_Target_Error` diagnostic.

- Added regression coverage for dry-run extraction through a symlinked output directory, ensuring the JSON report includes both `would-reject` and the precise `--output-dir path is unsafe` reason.

Build/test note: `gprbuild`, `gnatmake`, and `gnat1` remain unavailable in this environment, so this pass was completed by static inspection and source-level patching only.
